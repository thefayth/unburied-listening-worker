# Public / Private Boundary

## Public

- High-level description.
- Public-safe project story.
- Public-safe abstract visuals.
- Public-safe status.
- Non-sensitive roadmap.
- Public inquiry path through Faith's official channels.

## Private

- Source code unless Faith explicitly approves release.
- Agent instructions.
- Private prompts.
- Credentials, passcodes, tokens, and environment values.
- Adapters and private workflows.
- Private receipts.
- Customer, client, witness, family, child, legal, medical, benefits, admin, or
  evidence records.
- Raw transcripts, uploaded files, social-store records, watchlists, analysis
  artifacts, and storage data.
- Private screenshots.
- Strategy memos.
- Unpublished art/manuscripts unless explicitly chosen.
- Production systems.
- Deployment details.
- Financial details.
- Internal notes.

When in doubt, keep it private.
